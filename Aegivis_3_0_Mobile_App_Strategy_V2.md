# [개정판] Aegivis 3.0 현장 대응 지원 모바일 앱 (Safety-First & Site-Based Strategy)

본 문서는 사용자의 최신 피드백(실수 방지를 위한 현장 물리 버튼 조작 원칙)을 반영하여 전면 개정된 애플리케이션 개발 전략입니다.

---

## 🛡️ 1. 개발 철학: "원격 제어가 아닌, 현장 판단의 도구"

Aegivis 3.0 모바일 앱은 차단막을 직접 내리는 '리모컨'이 아닌, **현장 경찰이 가장 안전하고 빠르게 차단막을 내릴 수 있도록 돕는 '내비게이터 및 증거 채증 도구'**로 정의합니다.

*   **핵심 원칙**: 최종 물리적 차단은 현장 조작반(Button)에서만 가능.
*   **앱의 역할**: 골든타임 알림(Before-Flood) → 현장 출동 가이드(Navigation) → 물리 차단 지원(Manual Guide) → 조치 결과 보고(Logging).

---

## ✨ 2. 주요 핵심 기능 (Pivoted Features)

### **2-1. 지능형 조기 출동 알림 (AI Dispatch Alert)**
*   **기능**: 수위가 15cm에 도달하기 **15~20분 전**, 침수 확률과 예상 시간을 경찰/관리자에게 푸시 알림으로 전송.
*   **효과**: 경찰이 차에 타서 현장까지 이동하는 시간을 미리 확보하여, 물이 차기 전에 도로를 차단하게 함.

### **2-2. 다중 데이터 신뢰도 가이드 (Reliability Visualizer)**
*   **기능**: A, B, C, D 4단계 데이터 파이프라인의 일치도를 시각화.
*   **내용**: "레이더 수위와 기상청 예보가 95% 일치합니다. 현장 차단이 강력히 권고됩니다." 등의 텍스트 가이드 제공.
*   **효과**: 현장 경찰이 본인의 판단에 대한 기술적 확신을 가질 수 있게 함.

### **2-3. 현장 출동 내비게이션 연동**
*   **기능**: 알림 클릭 시, 현재 위치에서 해당 지하차도 진입로까지의 **최단 경로 내비게이션** 즉시 실행.
*   **효과**: 1분 1초가 급한 재난 상황에서 주소 입력 시간을 생략.

### **2-4. 물리 차단 체크리스트 (Safety Checklist)**
*   **기능**: 현장 도착 후 버튼을 누르기 전, 안전 확인 절차를 앱에서 가이드.
    *   [ ] 지하차도 내 잔류 차량 확인
    *   [ ] 진입 예정 차량 정지 신호 확인
    *   [ ] 물리 조작반 덮개 개방
*   **효과**: 긴박한 상황에서 발생할 수 있는 인적 오류(Human Error)를 체크리스트로 방지.

### **2-5. 조치 결과 채증 및 자동 보고 (Evidence Logging)**
*   **기능**: 물리 버튼을 누른 후, 차단막이 내려간 현장을 앱으로 촬영하면 즉시 보고서가 생성되어 관제 서버에 전송.
*   **효과**: 행정적인 사후 보고 절차를 간소화하고, 정확한 조치 시간을 기록.

---

## 🛠️ 3. 기술 아키텍처 및 보안

*   **데이터 통신**: **FastAPI + WebSockets** (실시간 데이터 동기화)
*   **알림 서버**: **FCM (Firebase Cloud Messaging)** + 자체 긴급 알림 엔진
*   **기록 관리**: 조치 완료 사진 및 데이터의 위변조 방지를 위한 타임스탬프 서버 운영.
*   **사용자 인증**: 지자체/경찰 전용 앱으로서 생체 인식(TouchID/FaceID)을 통한 보안 접속.

---

## 📅 4. 기대 효과

1.  **안전성 (Safety)**: 원격 해킹이나 시스템 오류로 인한 '뜬금없는 차단막 작동' 사고가 원천 차단됩니다.
2.  **신뢰성 (Legal Protection)**: "기계가 마음대로 내렸다"가 아니라 "현장 전문가가 기술적 근거를 바탕으로 직접 내렸다"는 논리로 법적 책임을 강화합니다.
3.  **효율성 (Efficiency)**: 경찰이 현장에 도착했을 때는 이미 AI가 모든 근거 데이터를 정리해 두었으므로, 육안 확인 후 1초 만에 결단을 내릴 수 있습니다.
